# Quick Fix Workflow

간단한 버그 수정이나 소규모 변경을 위한 경량 워크플로우입니다.

## 흐름

```
executor → reviewer
```

## 사용 조건

다음 조건을 **모두** 만족할 때만 사용:

| # | 조건 |
|---|------|
| 1 | 영향 파일 3개 이하 |
| 2 | 명확한 버그/수정 사항 |
| 3 | 설계 변경 없음 |
| 4 | 새로운 의존성 없음 |

**⚠️ 확실하지 않으면 `full-feature` 사용**

## 단계별 상세

### 1. Executor (필수)

**목적:** 버그 수정 / 소규모 변경

**에이전트:** `executor` (코드/씬/프리팹 모두 담당)

**체크리스트:**
- [ ] 문제 원인 파악
- [ ] 수정 구현
- [ ] 테스트 확인
- [ ] 컴파일 확인 (Unity)

**산출물:**
- 수정된 코드
- 커밋

**다음 단계:** reviewer

---

### 2. Reviewer (필수)

**목적:** 수정 검증

**체크리스트:**
- [ ] 수정이 문제를 해결하는지
- [ ] 다른 곳에 영향 없는지
- [ ] 테스트 통과

**산출물:**
- 간단한 승인/반려

**인라인 Retro:**
- 반복되는 버그면 기록
- 근본 원인 분석 필요시 기록

---

## 예상 소요

2단계 (반복 없으면)

## 격상 조건

작업 중 다음 상황 발견 시 `full-feature`로 격상:

| # | 상황 | 이유 |
|---|------|------|
| 1 | 영향 범위가 예상보다 큼 | 설계 검토 필요 |
| 2 | 근본 원인이 설계 문제 | architect 필요 |
| 3 | 새로운 패턴/구조 필요 | planner 필요 |
