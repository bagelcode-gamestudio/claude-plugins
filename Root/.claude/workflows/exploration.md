# Exploration Workflow

코드베이스 탐색, 구조 분석, 이해를 위한 워크플로우입니다.

## 흐름

```
architect (분석 모드)
```

## 사용 조건

| # | 상황 |
|---|------|
| 1 | 새 코드베이스 파악 |
| 2 | 특정 모듈 이해 |
| 3 | 의존성 분석 |
| 4 | 기술 부채 현황 파악 |

## 단계별 상세

### 1. Architect (분석 모드)

**목적:** 코드베이스 분석 및 문서화

**분석 유형:**

#### A. 전체 구조 분석
```markdown
## 분석 요청: 전체 구조

다음을 분석해주세요:
- 폴더 구조
- 주요 모듈
- 의존성 관계
- 진입점
```

#### B. 특정 모듈 분석
```markdown
## 분석 요청: [모듈명]

다음을 분석해주세요:
- 책임 범위
- 공개 API
- 내부 구조
- 의존성
```

#### C. 기술 부채 분석
```markdown
## 분석 요청: 기술 부채

다음을 분석해주세요:
- 중복 코드
- 복잡한 의존성
- 오래된 패턴
- 테스트 커버리지
```

**산출물:**
- 분석 결과 (대화 내)
- (선택) `docs/design/exploration-<주제>.md`
